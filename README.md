## النشر على GitHub Pages (الموصى به)

1. اعمل Repository جديد على GitHub (مثلاً `zayed-scout`).
2. ارفع **كل** الملفات والمجلدات زي ما هي (مهم: ملف `.nojekyll` لازم يترفع معاهم — هو اللي بيخلّي GitHub ميتجاهلش حاجة).
3. من الـ Repo: **Settings → Pages → Source: Deploy from a branch → Branch: main / (root) → Save**.
4. استنى دقيقة، GitHub هيديك رابط زي: `https://USERNAME.github.io/zayed-scout/`
5. افتح الرابط، وادخل بالباسورد المشترك.

> الرابط ده تقدر تبعته لكل القادة، يفتحوه من الموبايل أو الكمبيوتر على طول.

---

## التشغيل محلياً (لو حابب تجرّب قبل النشر)

> ⚠️ لا تفتح `index.html` بالدابل-كليك مباشرة — المتصفح بيمنع تحميل الأكواد كده. شغّله بسيرفر محلي:

- **ويندوز:** دابل-كليك على `تشغيل-ويندوز.bat`
- **ماك:** دابل-كليك على `تشغيل-ماك.command` (أول مرة: كليك يمين → Open → Open)
- **يدوياً:** افتح Terminal في المجلد واكتب `python -m http.server 8000` ثم افتح `http://localhost:8000`

## الجديد في النسخة دي

1. **توزيعة زايد الحقيقية** — صفحة القادة مبذورة بكل القادة والجوّالة الفعليين (٥١ فرد) من توزيعتكم، والقائد العام **ماريا فريد**. زرار «التوزيعة الرسمية» بيرجّع القايمة في أي وقت.
2. **القطاعات والجوّالة** — القادة موزّعين على القطاعات الثلاثة (متقدم · مبتدئ · أشبال) + العشيرة (الجوّالة)، زي توزيعتكم بالظبط.
3. **تقويم الموسم** — كل اجتماعات ومعسكرات وحفلات ٢٠٢٦ في تايملاين واحد بالتواريخ، مع فلاتر بالقطاع والنوع.
4. **المنهج والأهداف** — مرجع تعليمي كامل (كشفي · إبحار بحري · اجتماعي · بدني · ثقافي · روحي) لكل سنة في كل قطاع (١٦٨ هدف).
5. **المتابعة والتقييم** — تسجيل حضور (حاضر/بعذر/غائب) + درجات لكل اجتماع، وحساب نسبة الحضور ومتوسط الدرجات وتقدّم كل ولد.

## مشاركة البيانات بين القادة

> 📌 **مهم تفهمها:** البيانات (الأولاد، الحضور، تعديلات القادة) بتتحفظ في **متصفح كل قائد على جهازه** (LocalStorage)، مش على GitHub. يعني لو قائد ضاف أولاد على موبايله، القائد التاني مش هيشوفهم تلقائياً — حتى لو بيفتحوا نفس الرابط.

عشان تشاركوا البيانات:

- **القادة:** صفحة القادة → «تصدير» → ملف JSON → ابعته للقائد التاني → عنده «استيراد».
- **الأولاد:** نفس الكلام في صفحة الأولاد.
- **المتابعة:** نفس الكلام في صفحة المتابعة.

> لو عايزين بيانات مشتركة حقيقية بين كل القادة (بدون تصدير/استيراد يدوي)، ده محتاج قاعدة بيانات أونلاين (زي Firebase) — نقدر نضيفها لو احتجتوها لاحقاً.
